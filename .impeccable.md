# Design Context

> Source of truth for all design decisions on the Marble Skill Taxonomy Explorer.
> Captured via `/impeccable teach`. Every future design change should be checked
> against the Principles below before proceeding.

## Product

Marble Skill Taxonomy — an open dataset of 1,590 learning concepts (ages 4–15)
across 8 subjects, visualized as an interactive 3D graph. The explorer is a
single-page React 19 + Vite + Three.js + Tailwind v4 app. One route; the 3D
canvas is the centerpiece, with a tabbed sidebar (Inspect / Catalog / Standards
/ Pathway) and floating overlays.

## Users

**Parents & families.** They arrive curious about what their child is learning
— not as curriculum experts. Their context: a quiet moment on a laptop or
phone, wanting reassurance and clarity, not a database query interface. They
should leave feeling *“so that’s what all of this is”*, not overwhelmed.

Secondary audiences (educators, EdTech builders, researchers) use the same UI
but the layout prioritizes the parent: legibility over density, plain labels
over jargon, and a graph that earns attention before the chrome demands any.

## Brand Personality

**Calm. Precise. Reassuring.** Like a museum exhibit caption placed beside a
luminous specimen — authoritative without being cold, inviting without being
childish. The voice is confident and plain. It does not try to *seem*
educational; it simply is.

Three-word personality: **luminous · precise · quiet**.

## Aesthetic Direction

**A quiet instrument.** The 3D graph is the specimen: it carries all the color,
luminosity, and wonder (planetarium energy through light and motion, not
saturation). The surrounding UI is the caption: near-monochrome, restrained,
stepping back so the graph is always the focal point.

### References
- **Observatory / scientific instrument** — dark, luminous, calibrated. The
  hush of a planetarium or a well-made star map. Authority through precision.
- **Editorial atlas** — printed-reference discipline. Strict grid, restrained
  color, serif headings that signal “this is a curated source.”
- **Museum exhibit caption** — warm, human, generous space. Legible type that
  reassures a non-expert they’re in good hands.

### Anti-references (must NOT look like)
- **AI EdTech slop** — stock illustration, rounded gradient cards, cartoon
  mascots, busy chrome, the generic “learning platform” template.
- Also avoid: glow-for-glow’s-sake, glassmorphism as decoration, gradient text,
  colored side-stripes on cards, sparkline decoration, anything that competes
  with the graph for attention.

### Theme
**Dark, refined.** Near-black canvas with a subtle hue tint (not pure gray).
Light type. The graph glows against it; the UI recedes.

### Color discipline
**Near-monochrome UI.** The UI chrome is mostly tinted neutrals + a single
luminous accent. The graph’s subject palette (a muted diverging set: cold blues
→ sky blue → cream → muted rose) carries all the hue in the product. The UI
does not introduce its own color — it frames the graph’s color.

- Use OKLCH for all values. Tint neutrals toward the brand hue.
- One accent, used rarely (60-30-10 visual weight).
- No pure black/white; always tint.

### Typography direction
> ⚠️ **Open item.** The current working tree loads Inter (body), Space Grotesk
> (display), JetBrains Mono (mono), and Fraunces (serif) — **all four are on
> impeccable’s banned monoculture list.** A font refresh is future work; until
> then, this section records the *target* discipline, not the current fonts.

Target: pair a distinctive display face with a refined body face, chosen via
impeccable’s font-selection procedure (brand words: *luminous · precise ·
quiet*). Reserve monospace for genuine tabular data (ages, counts, IDs) only —
never as decoration. Cap body line length at 65–75ch. Fixed-rem scale for this
app/dashboard UI (no fluid type in product UI).

## Design Principles

1. **The graph is the specimen; the UI is the caption.** The 3D graph carries
   all color, luminosity, and wonder. The surrounding UI stays near-monochrome
   and quiet, stepping back so the graph is always the focal point. If a UI
   element competes with the graph for attention, it loses.

2. **Calm over spectacle.** Planetarium wonder comes from luminosity and
   motion, not saturation or chrome. Every UI element earns its place by being
   useful — never decorative. When in doubt, remove.

3. **Reassuring for non-experts.** A parent should never feel lost. Labels are
   plain, hierarchy is obvious, empty states teach the interface, and density
   never overwhelms. Prefer one clear path over five clever ones.

4. **Instrument precision.** Type, spacing, and color follow a strict,
   calibrated system: one accent, tinted neutrals, a modular scale. No AI-slop
   tells (no gradient text, no side-stripe borders, no glassmorphism-for-its-
   own-sake, no sparkline decoration).

5. **Accessibility is not optional.** The dark theme and heavy 3D canvas must
   remain usable for everyone: `prefers-reduced-motion` honored throughout,
   full keyboard navigation, ARIA semantics preserved, color contrast at AA or
   better. The graph is wondrous; the UI must be usable.
